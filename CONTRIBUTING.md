# Contributing Guidelines

Thank you for your interest in contributing to this project!  
This repository is designed for educational, analytical, and portfolio purposes.  
Contributions are welcome and encouraged.

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button on GitHub to create your own copy of this repo.

### 2. Create a Feature Branch
Use a branch name that describes your change:
git checkout -b feature/improve-model
### 3. Make Your Changes
Examples of contributions:
- Improving notebooks (EDA, ML, segmentation)
- Adding new visualizations or insights
- Improving Power BI dashboard layout
- Documentation improvements
- Performance optimization

### 4. Commit Your Changes
git commit -m "Improved churn model and added cross-validation"
### 5. Push and Submit a Pull Request
git push origin feature/improve-model
Then open a **Pull Request** on GitHub describing the change.

## Guidelines

- Follow clear naming conventions.
- Keep code readable and well-commented.
- Avoid uploading large raw datasets.
- Ensure Power BI files do not exceed GitHub limits.
- Be respectful in communication.

Thanks again for contributing!
